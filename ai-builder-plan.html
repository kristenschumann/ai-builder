Cloud Email Manager
A fully automated daily email triage system that reads, categorizes, drafts replies in my voice, and sends me a Slack summary — no laptop required.
Cloud Email Manager

What I built

Automation flow
1
Schedule Trigger — daily at 4pm
Kicks off automatically every day at 4pm MT. No manual trigger, no laptop needed.
2
Get many messages
Pulls the last 24 hours of Gmail messages via OAuth2.
3
Key Fields + Remove Extra Text
Extracts sender, subject, and body. Strips signatures, footers, and threading noise to reduce token cost.
4
Email Categorizer — Claude Haiku
Fast, cheap categorization pass. Each email gets labeled: Noise, Informational, Needs Reply, or Urgent. Noise and Informational are dropped here.
5
If: Needs Reply or Urgent
Branching logic routes action-needed emails to the drafter. Everything else stops here.
6
Email Drafter Agent — Claude Sonnet + Gmail tool
Sonnet writes a reply draft in my voice and saves it directly as a Gmail draft. Nothing is sent automatically.
7
Summarize + Slack DM
Sends me a Slack DM with what was drafted, what was flagged urgent, and one-click links to each Gmail draft for review.
How it decides what gets a draft

Haiku categorizes each email into one of four buckets based on a system prompt that defines my priorities. Noise is anything automated, promotional, or newsletter-style. Informational is genuine but requires no action — FYIs, confirmations, receipts. Both are discarded.

Needs Reply triggers a draft: real human senders asking a question, requesting input, or waiting on me to move something forward. Urgent also triggers a draft and additionally gets flagged in the Slack summary with a priority marker, so I know to review it first.

Cloud Email Manager

Documentation

Tools used
n8n Cloud Gmail OAuth2 Claude Haiku (categorizer) Claude Sonnet (drafter) Slack OAuth2
How to deploy
1.Set up an n8n Cloud account and create a new workflow.
2.Add a Schedule Trigger node set to run daily at your preferred time.
3.Connect your Gmail account via OAuth2. Add a "Get Many Messages" node filtered to the last 24 hours.
4.Add a "Set" node to extract key fields (from, subject, body) and a Code node to strip signatures and thread noise.
5.Add an Anthropic LLM node (Haiku model) with the Email Categorizer system prompt below. Parse the JSON output.
6.Add an IF node routing emails categorized as "needs_reply" or "urgent" to the next step; discard the rest.
7.Add a second Anthropic LLM node (Sonnet model) as an agent with Gmail draft tool access. Use the Email Drafter system prompt below.
8.Connect your Slack account via OAuth2. Add a Slack node to DM yourself the summary and draft links.
9.Activate the workflow. It runs on schedule with no laptop required.
Email Categorizer system prompt (Haiku)
You are an email categorizer. Your job is to read an email and classify it into exactly one of four categories. Categories: - noise: Automated emails, newsletters, promotional content, marketing, notifications, receipts, confirmations that require no response. - informational: Genuine human-sent emails that are FYIs, updates, or context — no reply is needed or expected. - needs_reply: A real person is asking me a question, waiting on my input, or the email will stall something if I don't respond. - urgent: Same as needs_reply but time-sensitive — needs a response today or there's a meaningful consequence to waiting. Rules: - When in doubt between needs_reply and informational, choose needs_reply. - Automated system emails are always noise, even if they mention my name. - Be conservative with urgent — only use it when there's a genuine time constraint or high stakes. Respond with JSON only. No explanation. Format: { "category": "noise" | "informational" | "needs_reply" | "urgent", "reason": "one short sentence explaining the classification" }
Email Drafter Agent system prompt (Sonnet)
You are an email drafting assistant for Kristen Hansen, GM of Courses at Leland. Your job is to write reply drafts to emails that need a response. You will receive the original email and should draft a reply in Kristen's voice. Voice and style: - Casual and direct. No fluff, no unnecessary pleasantries. - Short. Get to the point immediately. - Confident tone. No hedging. - No em dashes. - Strong verbs. Cut filler words. - Do not start with "I hope this finds you well" or any opener like it. - Match the register of the original email — if it's casual, stay casual. Instructions: - Draft only the reply body. Do not include a subject line unless explicitly requested. - If a question is asked, answer it clearly and concisely. - If something is being requested, either commit to it with a timeframe or push back with a reason. - Do not speculate on details you don't know — leave a [placeholder] if context is missing. - Keep drafts under 5 sentences unless the email genuinely requires more. After writing the draft, use the Gmail tool to save it as a draft in Kristen's Gmail account, replying to the original thread.
